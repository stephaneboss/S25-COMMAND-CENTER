# Smajor System Axes

## But

Rendre explicite la structure reelle de `Smajor` pour que:
- le site
- l'admin
- les agents
- le backend S25

parlent tous du meme systeme.

## Axe 1 - Business terrain

Scope:
- excavation
- deneigement
- multi services exterieurs
- clients actifs
- interventions terrain
- camions, pelle, equipes, facturation

But:
- recevoir une demande
- qualifier
- planifier
- envoyer les hommes
- executer
- facturer
- cloturer

## Axe 2 - Administration temps reel

Scope:
- identites
- organisations
- employes
- fournisseurs
- roles
- services actives
- acces portail
- dispatch
- suivi financier et audit

But:
- savoir qui existe dans le systeme
- savoir qui a acces a quoi
- savoir quel service est actif pour qui
- garder une chaine stricte et lisible

## Axe 3 - Backend IA / S25

Scope:
- cockpit S25
- missions
- intel
- MERLIN MCP
- TRINITY
- COMET
- mesh Akash

But:
- orchestrer
- memoriser
- valider
- surveiller
- stabiliser l'infra
- soutenir le business sans le remplacer

## Regle

1. Le business terrain reste la raison d'etre.
2. L'administration stricte controle les acces et les flux.
3. Le backend IA sert les deux autres axes.
4. Aucune de ces couches ne doit devenir un silo isole.
